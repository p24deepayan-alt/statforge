Quickstart Guide

Welcome to StatForge. This guide will walk you through the process of establishing a connection to your data warehouse, running your first preprocessing pipeline, and generating a baseline statistical report. StatForge is designed to handle high-density datasets with precision and minimal configuration.

1. Initializing the Environment

Before executing any models, you must initialize the StatForge core environment. This step allocates necessary memory limits and establishes the execution context for your analytical workflows.

python 
import statforge as sf

# Initialize with standard memory constraints
env = sf.init_environment(
    memory_limit="16GB",
    precision="float64",
    logging_level="INFO"
)

print(f"StatForge initialized: {env.status}")

2. Importing Datasets

StatForge supports zero-copy loading for Parquet and Arrow formats, ensuring optimal performance for large analytical workloads. For this example, we will load a standard CSV dataset into a defined DataFrame schema.

info
Data Integrity Note

When utilizing CSV imports, StatForge will automatically attempt to infer schema types based on the first 10,000 rows. For maximum reliability in production environments, explicitly define your schema definitions prior to ingestion.

3. Basic Profiling

Once data is loaded, utilize the profile_metrics() function to generate a comprehensive overview of variable distributions, missing values, and high-correlation pairs.

Parameter Type Description
target_variable string The primary column for dependency analysis.
compute_quantiles boolean If true, calculates Q1, Median, and Q3 for all numeric columns.
outlier_threshold float Z-score threshold for marking outliers. Default is 3.0.
